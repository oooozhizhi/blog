    @411www.lintcode.com/zh-cn/problem/gray-code


    题目：411. 格雷编码
    描述：
        格雷编码是一个二进制数字系统，在该系统中，两个连续的数值仅有一个二进制的差异。给定一个非负整数 n ，表示该代码中所有二进制的总数，请找出其格雷编码顺序。一个格雷编码顺序必须以 0 开始，并覆盖所有的 2n 个整数。


    注意事项
        对于给定的 n，其格雷编码顺序并不唯一。
        根据以上定义， [0,2,3,1] 也是一个有效的格雷编码顺序。
    样例
        给定 n = 2， 返回 [0,1,3,2]。其格雷编码顺序为：
        00 - 0
        01 - 1
        11 - 3
        10 - 2


    @1www.lintcode.com/zh-cn/problem/a-b-problem


    题目:1. A + B 问题
    描述：给出两个整数a和b, 求他们的和, 但不能使用 + 等数学运算符。


    说明：
    a和b都是 32位 整数么？
            是的
    我可以使用位运算符么？
            当然可以
    样例？
            如果 a=1 并且 b=2，返回3


    思路：
    加法:对于两个数a和b，我们分别计算add = a^b表示本次留下来的bit位， carry=(a&b)<<1 表示发生进位的数。一直重复这样的计算，直到carry=0停止。
    减法:同于加法,公式为a-b = a + (~b+1)
    限于时间，乘法,除法贴出我觉得好的网站:https://blog.csdn.net/ojshilu/article/details/11179911

    @181www.lintcode.com/zh-cn/problem/flip-bits


    题目:181. 将整数A转换为B
    描述：如果要将整数A转换为B，需要改变多少个bit位？


    样例
    如把31转换为14，需要改变2个bit位。
    (31)10=(11111)2
    (14)10=(01110)2


    思路：简单的想法就是看a 和 b 有多少不同位
    a|b记录a和b并
    a&b记录a和b交
    (a|b) ^ (a&b)记录a和b有多少位不同，最后再计算有多少位上是1就行了
    注意：1. JAVA的负数编码问题：假设a是一个正数，那么有-a=(~a+1)

    @180www.lintcode.com/zh-cn/problem/binary-representation


    题目:180. 二进制表示
    描述：给定一个数将其转换为二进制（均用字符串表示），如果这个数的小数部分不能在 32 个字符之内来精确地表示，则返回 “ERROR”。


    样例
    n = “3.72”, 返回 “ERROR”.
    n = “3.5”, 返回 “11.1”


    思路:
    1 整数的编码方式就是常见的二进制编码
    2 小数部分的二进制编码方式为：假设a<1,如果2a>1，该位编码1，并且a -= 1.如果2a<1,该位编码0.不断重复，直到达到位数的限制。

    @723www.lintcode.com/zh-cn/problem/rotate-bits-left


    题目:723. 左旋转二进制位
    描述：
    位旋转 -—— 旋转（或循环移位）是类似于移位的操作, 不同的是一端脱落的那一位会被放回到另一端
    在左旋转中, 左端掉下来的那一位会放在右端
    假设 n 用 8 位二进制来存. 对 n = 11100101 左旋转 3 位, 得到 n = 00101111 (左移3位, 原先的前3位放在末尾).
    如果 n 用 16 位或 32 位二进制来存, 那么对 n (000…11100101)左旋转了之后会变成 (00..0011100101000).
    在本问题中, 你可以假设 n 是用 32 位二进制来存的.


    样例
    给出 n = 123, d = 4 返回 183


    思路：来一个mask变量探测最左端的是1还是0，然后拿一个 变量记录结果，最后把这两个变量整合到一起就行了。
