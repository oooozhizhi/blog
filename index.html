约瑟夫问题

昨天做笔试题的时候遇到这个问题，觉得有很意思，在这里记录一下。
约瑟夫问题：

  • 所有人围成一圈

  • 顺时针报数，每次报到q的人将被杀掉

  • 被杀掉的人将从房间内被移走

  • 然后从被杀掉的下一个人重新报数，继续报q，再清楚，直到剩余一人


    • 输入：一共有n个人，每次报到k的时候，那个人被杀掉
    初始情况：0,1,2…n-2,n-1(共n个人)
    假设第一个人已经自杀，剩下的n-1个人组成了一个新的约瑟夫环(以编号为k%n)的人开始：k k+1 k+2 … n-2 n-1 0 1 2 … k-3 k-2 对剩余的人编号做一下转换：


    x –> x’ (这里的x表示的是最后留下来那个人x’就是我们要求的结果)
    k –> 0
    k+1 –> 1
    …
    k-2 –> n-2
    k-1 –> n-1


            变换之后就成了n-1个人报数的子问题，假如我们知道这个子问题的解(也就是x的坐标),那么我们就可以知道原问题的解。可以知道转换公式为:

    x(n) = (x(n-1) + k)%n

            根据上面的公式，我们可以从只有一个人开始坐标转换，最后到第n轮该人的坐标就是我们要的结果。

    二进制

        做了很久时间的题目，感觉各个部分的知识非常的杂乱，趁现在有时间对知识进行归档工作。这篇文章是关于二进制的题目，有时候会遇到二进制相关的题目，感觉里面的trick还挺多的，这里开一个头，之后遇到二进制相关的工作会陆续归档到这篇文章里面。
        下面是LintCode上遇到的一些题目：


    365.二进制中有多少个1
    408.二进制求和
    411.格雷编码 <*>


    365/408，利用一般的思路就能够将题目解完。
    411：利用递进的思路，比如说：
    n=1 的时候，只有两种可能{0,1}
    n=2 的时候, 需要在n=1的结果上再左边补上一个0/1,此时的结果为:{0{0,1},1{1,0}}={00,01,11,10}
    n=3 的时候，需要基于n=2的结果,此时的结果为:{0{00,01,11,10}(n=2的正向结果),1{10,11,01,00}(n=2的反向结果)}={…}

    样例：

        给定 5 (101)，返回 2
        给定 1023 (1111111111)，返回 10

    Tips:
    关于如何数int中1的个数，这篇文章讲的很好：https://www.cnblogs.com/graphics/archive/2010/06/21/1752421.html

    @408www.lintcode.com/zh-cn/problem/add-binary


    题目：408. 二进制求和
    描述：
        给定两个二进制字符串，返回他们的和（用二进制表示）。


    样例：
        a = 11
        b = 1
        返回 100

    @411www.lintcode.com/zh-cn/problem/gray-code


    题目：411. 格雷编码
    描述：
        格雷编码是一个二进制数字系统，在该系统中，两个连续的数值仅有一个二进制的差异。给定一个非负整数 n ，表示该代码中所有二进制的总数，请找出其格雷编码顺序。一个格雷编码顺序必须以 0 开始，并覆盖所有的 2n 个整数。


    注意事项
        对于给定的 n，其格雷编码顺序并不唯一。
        根据以上定义， [0,2,3,1] 也是一个有效的格雷编码顺序。
    样例
        给定 n = 2， 返回 [0,1,3,2]。其格雷编码顺序为：
        00 - 0
        01 - 1
        11 - 3
        10 - 2

    @224leetcode.com/problems/basic-calculator/description

    题目(+ - () 计算器)


    Implement a basic calculator to evaluate a simple expression string.
    The expression string may contain open ( and closing parentheses ), the plus + or minus sign -, non-negative integers and empty spaces .
    You may assume that the given expression is always valid.
    Some examples:
    “1 + 1” = 2
    “ 2-1 + 2 “ = 3
    “(1+(4+5+2)-3)+(6+8)” = 23

    解法：

    2018-03-21
    rewrite(3) Deng
    Time O(N) Space O(N)
    idea:
    主要的思路是开辟两个栈：一个符号栈，一个数字栈。
        1 利用Character.isDisgit() 来判断一个字母是不是属于数字，如果是数字的话通过：
                 num = 10 * num + (char - ‘0’)
        的方式来获得多位的数字。
         2 题目中只有 ‘空格’ ‘ + ‘ ‘ - ‘ ‘(‘ ‘ ) ‘ 四种符号，并且式子都是有意义的。为了保证 + - 号能够正确的计算，每进来一个新符号我们都会计算完之前栈中的结果。
         3 最后结束的时候要考虑多种边界条件。

    @227leetcode.com/problems/basic-calculator-ii/description

    题目 (+ - * / 计算器)


    Implement a basic calculator to evaluate a simple expression string.
    The expression string contains only non-negative integers, +, -, , / operators and empty spaces . The integer division should truncate toward zero.
    You may assume that the given expression is always valid.
    Some examples:
    “3+2    2” = 7
    “ 3/2 “ = 1
    “ 3+5 / 2 “ = 5

    解法：

    2018-03-21
    rewrite(3) Deng
    Time O() Space O()
    idea:
        跟上面的思路相同，不过引入了两个新的计算符号 ‘*’ ‘/‘ 要注意出现这两个符号要优先计算。不过没有了括号，难度跟上面一致。

    前缀表达式-中缀表达式-后缀表达式

        上面的两个式子都是中缀表达式的计算方式。实际上四则运算中，前缀表达式，中缀表达式，后缀表达式都是四则运算的表达方式，用以四则运算的表达式求值。
        1 中缀表达式:(3+4)5-6 其实就是常见的运算表达式。
        2 前缀表达式又称波兰式，前缀表达式的运算符位于操作符之前。比如：-    +3456。可以很容易计算出前缀表达式的值。
        3 后缀表达式又称波兰表达式，与前缀表达式类似，只是运算符位于操作数之后。比如:3 4 + 5 * 6 -

    之后将会补充各个表达式之间互相转换的代码。

    图论-最短路

    之前就想对常见的图论的算法进行一个总结，借着这个机会开一个坑。

    常见的最短路算法

    floyd 弗洛伊德算法

    思想：想知道a到b点的最短路，只能加入中间节点来缩短路径。那么我们就要遍历所有可能的中间点，才能得到a到b的最短路径。
    核心代码只有5行

    for(int mid = 0;mid < N;mid++){
    for(int i=0;i < N;i++){
        for(int j=0;j < N;j++){
        if(map[i][j] > map[i][k] + map[k][j])    map[i][j]=map[i][k]+map[k][j];
        }
    }
}

    这种算法可以找多源多汇最短路。
    时间复杂度 O(N^3)
    空间复杂度 O(N^2)

    dijkstra 狄杰斯特拉 － 邻接矩阵

    思想：这个算法只能计算单源最短路，而且不能计算负权值，这个算法是贪心思想。
    e[N][N]邻接矩阵 dis[N]初始点到个点的距离 book[N]标记该点最短路是否找到 假设每个量都被正确初始化了。

    for(int times=1;times < N;times++){
    int min = Integer.MAX_VALUE;
    int index = -1;
    //寻找本次最短路，标记为最终最短路
    for(int i=0;i < N;i++){
        if(!book[i] &&dis[i] < min){
            min = dis[i];
            index = i;
        }
    }
    if(index == -1)    break;//提前退出
    book[index] = true;
    for(int j=0;j < N;j++){
        if(e[i][j] == Integer.MAX_VALUE){
            if(dis[j] > dis[index] + e[index][j])    dis[j] = dis[index] + e[index][j];
        }
    }
}

    dijkstra 狄杰斯特拉 － 邻接表 ＋ 优先队列

    例题：leetcode－744
    输入：
    times[i] = (u, v, w) u源点v汇点w权重
    N 网络节点总数
    K 起始点
    时间复杂度 O(N^2)
    空间复杂度 O(N logN)

    public int networkDelayTime(int[][] times,int N,int K){
    //开空间,用邻接表的方式存储图
    Map<Integer,List<int[]>> graph = new HashMap<>();
    //优先队列 <dist , end node>
    PriorityQueue<int[]> heap = new PriorityQueue<int[]>((info1,info2)->info1[0] - info2[0]);
    //距离矩阵 <end node , dist>
    Map<Integer,Integer> dist = new HashMap<>();
    //存入数据
    for(int[] edge: times){
        if(!graph.containsKey(edge[0])){
            graph.put(edge[0],new ArrayList<int[]>());
        }
        graph.get(edge[0]).add(new int[]{edge[1],edge[2]});
    }
    //存入初始点，从K点出发，
    heap.offer(new int[]{0,K});
    while(!heap.isEmpty()){
        int[] info = heap.poll();
        int d = info[0], node = info[1];
        //如果包含这个点continue | 先到先的,因为优先队列能够保证最后的结果是
        if(dist.containsKey(node))    continue;
        //如果出现新的点，放入最后的距离队列
        dist.put(node,d);
        //如果图中包含该点到其他点的边
        if(graph.containsKey(node)){
        //对于出发点的所有边
            for(int[] edge : graph.get(node)){
                int nei = edge[0], d2 = edge[1];
                //如果dist中不包含这个尾部节点
                if(!dist.containsKey(nei))
                    heap.offer(new int[]{d+d2,nei});
            }
        }
    }
    if(dist.size() != N)    return -1;
    int ans = 0;
    for(int cand : dist.values())
    ans = Math.max(ans,cand);
    return ans;
}
