Volatile使用

Java语言中的volatile变量可以被看作一种”程度较轻的synchronized”;与synchronized块相比，volatile变量所需的编码较少，并且运行时开销也较少，但是它所能实现的功能也仅是synchronized的一部分。本文只简要的介绍一下volatile变量，具体内容请看扩展。


@正确使用Volatile变量www.ibm.com/developerworks/cn/java/j-jtp06197.html

锁提供了两种主要特征：互斥(mutual exclusion)和可见性(visiblility)。互斥即一次只允许一个线程持有某个特定的锁，因此可使用该特性实现对共享数据的协调访问协议，这样，一次就只有一个线程能够使用该共享数据。可见性要更加复杂一点，它必须确保释放锁之前对共享数据做出的更改对于随后获得该锁的另一个线程是可见的。

Volatile变量

Volatile变量具有synchronized的可见性特征，但是不具备原子特性。这就是说线程能够自动发现volatile变量的最新值。Volatile变量可用于提供线程安全，但是只能应用于非常有限的一组用例：多个变量之间或者某个变量的当前值与修改后值之间没有约束。因此，单独使用volatile还不足以实现计数器，互斥锁或任何具有与多个变量相关的不变式的类。
出于简易性或可伸缩的考虑，您可能倾向于使用volatile变量而不是锁。当使用volatile变量而非锁时，某些习惯用法更加易于编码和与阅读。此外，volatile变量不会像锁那样造成线程阻塞，因此也很少造成可伸缩性问题。在某些情况下，如果读操作远远大于写操作，volatile变量还可以提供优于锁的性能优势。

正确使用Volatile变量的条件

只能在有限的一些情形下使用volatile变量代替锁。要使volatile变量提供理想的线程安全，必须同时满足下面两个条件：

  • 对变量的写操作不依赖于当前值

  • 该变量没有包含在具有其他变量的不变式中


    • 实际上，这些条件表明，可以被写入volatile变量的这些有效值独立于任何程序的状态，包括变量的当前状态。
    第一个条件的限制使volatile变量不能用作线程安全计数器。虽然增量操作看上去类似一个单独操作，实际上它是一个由读取-修改-写入操作序列组成的组合操作，必须以原子方式执行，而volatile不能提供必须的原子特性。实现正确的操作需要使x的值在操作期间保持不变，而volatile变量无法实现这点。

    性能考虑

    使用volatile变量的主要原因是其简易：在某些情况下，使用volatile变量要比使用相应的锁简单的多。使用volatile变量次要原因是其性能：某些情况下，volatile变量同步机制的性能要优于锁。
    在目前大多数的处理器架构上，volatile读操作开销非常低–几乎和非volatile操作一样。而volatile写操作的开销要比非volatile写操作多很多，因为要保证可见性需要实现内存界定。即便如此，volatile的总开销仍然要比锁获取低。
    volatile操作不会像锁一样造成阻塞，因此，在能够安全使用volatile的情况下，volatile可以提供一些优于锁的可伸缩特性。如果读操作的次数要远远超过写操作，与锁相比，volatile变量通常能够减少同步的性能开销。

    结束语

    与锁相比，Volatile变量是一种非常简单但同时又非常脆弱的同步机制，它在某些情况下将提供优于锁的性能和伸缩性。如果严格遵循Volatile的使用条件–即变量真正独立于其他变量和自己以前的值–在某些情况下可以使用volatile代替synchronized来简化代码。然而，使用volatile的代码往往比使用锁的代码更加容易出错。

    其实里面重复的内容还挺多的，这里我就统一贴上线段树的代码，包含构造，查询，修改。其他的只需要根据实际问题不同，修改对应的构造，查询方式就可以了。

    对于线段树中的每一个非叶子节点[a,b]，它的左儿子表示的区间为[a,(a+b)/2]，右儿子表示的区间为[(a+b)/2+1,b]。因此线段树是平衡二叉树，最后的子节点数目为N，即整个线段区间的长度。
    使用线段树可以快速的查找某一个节点在若干条线段中出现的次数，时间复杂度为O(logN）。而未优化的空间复杂度为2N，因此有时需要离散化让空间压缩。
    线段树对于修改的时间复杂度也是log(N),这样在查询很多的场景中，性能还是挺好的。

    JAVA基础知识

    这里简要的记录一些我不是很清楚的JAVA知识，主要是一些语法相关的知识。至于多线程相关的知识，看我进度吧。
    1数据类型：
    1)基本数据类型(8种):byte,short,int,long,float,double,char,boolean。这种数据类型是赋值。
    2)引用数据类型(3种):数组，类，接口。这种数据类型是引用。
    2如何在java中定义一个数组：
    1)元素类型[] 变量名 = new 元素类型[元素个数];
    2)元素类型[] 变量名 = {元素1,元素2,…};
    3)元素类型[] 变量名 = new 元素类型[]{元素1,元素2,…}
    3java分了5片内存
    3.1.寄存器
    3.2.本地方法区
    3.3.方法区
    3.4.栈
    1)栈中存储的都是局部变量(函数中定义的变量，函数上的参数，语句中的变量)，只要数据运算完成所在的区域释放。
    3.5.堆
    1)用于存储数组和对象，也就是实体。
    1.1)每个实体都有内存首地址
    1.2)堆内存中的变量都有默认初始化值，因为数据类型不同，值也不一样。
    1.3)垃圾回收机制。
    4类中怎么没有定义主函数？
    1)主函数的存在，仅为该类是否需要独立运行，如果不需要，主函数是不用定义的。
    2)主函数的解释:保证所在类的独立运行，是程序的入口，被jvm调用。
    5_创建一个对象在内存中做了什么事情？Person p = new Person()
    1)先将硬盘上指定位置的Person.class文件加载进内存。
    2)执行main方法时，在栈内存中开辟了main方法的空间(压栈->进栈)，然后在main方法的栈区分配了一个变量p。
    3)在堆内存中开辟了一个实体空间，分配了一个内存首地址值。new
    4)在该实体空间中进行属性的空间分配，并进行了默认初始化。
    5)对空间中的属性进行显示初始化。
    6)进行实体的构造代码块初始化。
    7)调用该实体对应的构造函数，进行构造函数的初始化。
    8)将首地址赋值给p,p变量就引用了该实体。（指向该对象）
    6.static:关键字，是一个修饰符，用于修饰成员
    特点：
    1.想要实现对象中的共性数据的对象共享。可以将这个数据进行静态修饰。
    2.被静态修饰的成员，可以直接被类名所调用。也就是说，静态的成员多了一种调用方式。类名.静态方式
    3.静态随着类的加载而加载，而且优先于对象存在。
    弊端:
    1.有些数据是对象特有的数据，是不可以被静态修饰的。
    2.静态方法只能访问静态成员，不可以访问非静态成员。因为静态方法加载时，优先于对象存在，所有没有办法访问对象中的成员。
    7.单例设计模式
    1.解决的问题：保证一个类在内存中的对象唯一性。比如：多程序读取一个配置文件时，建议配置文件封装成对象。会方便操作其中数据，又要保证多个程序读到的是同一个配置文件对象，就需要该配置文件对象在内存中是唯一的。
    2.如何保证对象的唯一性？
    1)不让其他程序创建该类的对象。
    2)在本类中创建一个本类对象。
    3)对外提供方法，让其他程序获取这个对象。
    4)步骤
    4.1)因为创建对象都需要构造函数初始化，只要将本类中的构造函数私有化，其他程序就无法再创建该类。
    4.2)就在类中创建一个本类的对象。
    4.3)定义一个方法，返回该对象，让其他程序可以通过方法就能够得到本类对象。这里有两种方式，一种是定义成static，这样这些信息随着类加载的时候就加载进来了。另外一种方式是，等到外部访问这类对象的时候，该对象再加载。
    9.final关键字
    1)这个关键字是一个修饰符，可以修饰类，方法和变量。
    2)被final修饰的类是一个最终类，不可以被继承。
    3)被final修饰的方法是一个最终方法，不可以被覆盖。
    4)被final修饰的变量是一个常量，只能赋值一次。
    抽象类:abstract
    1)抽象方法只能定义在抽象类中，抽象类和抽象方法必须由abstract关键字修饰。
    2)抽象方法值定义方法声明，并不定义方法实现。
    3)抽象类不可以被创建对象(实例化)。
    4)只用通过子类集成并覆盖了抽象类中所有抽象方法后，该子类才可以实例化。
    5)抽象类的细节：
    5.1)抽象类中是否有构造函数？有，用于给予抽象类对象进行初始化。
    5.2)抽象类中是否可以定义非抽象方法？可以，其实，抽象类和一般类没有太大的区别，都是在描述事务，只不过抽象类在描述事物时，有些功能不具体。
    5.3)抽象关键字abstract和哪些不可以共存?final,private,static
    10.接口
    1)是用关键字interface定义的。
    2)接口包含的成员，最常见的有全局常量，抽象方法。
    2.1)接口的成员都有固定的修饰符。成员变量:public static final 成员方法:public abstract
    3)接口中有抽象方法，说明接口不可以实例化。
    4)类与类之间存在着继承关系，类与接口中间存在的是实现关系。继承用extends,实现用implements
    5)一个类在继承另一个类的同时，还可以实现多个接口。所以接口的出现避免了单继承的局限性。
    6)其实java中是有多继承的。接口与接口之间存在着继承关系，接口可以多继承接口。
    11.如果想用子类对象的特有方法，如何判断对象是哪个具体的子类类型
    1)可以通过一个关键字instanceof，格式<对象 instanceof 类型>，判断一个对象是否所属于指定的类型。举例：Student instanceof Person = true;//student类继承了person类
    12.异常
    1)定义：就是不正常，程序在运行时出现的不正常的情况。其实就是程序中出现的问题。这个问题按照面向对象思想进行描述，并封装成了对象。因为问题的产生有产生的原因，有问题的名称，有问题的描述等多个属性信息存在。当出现多属性信息最方便的方式就是将这些信息进行封装。异常类：java.lang.Throwable;
    2)Throwable是可抛出的。Error，错误一般情况下，不编写针对性的代码进行处理，通常是jvm发生的，需要对程序进行修正。Exception,异常，可以有针对性的处理方式。
    3)对于捕捉，java有针对性的语句块进行处理：
    try{需要被检测的代码;}catch(异常类 变量名){异常处理代码;}finally{一定会执行的代码;}
    4)throw和throws关键字的区别：throw用于抛出异常对象，后面跟的是异常对象，throw用在函数内；throws用于抛出异常类，后面跟的异常类名，throws用在函数上。
    5)异常分为两种：
    5.1)编译时被检查的异常，只要是Exception及其子类都是编译时被检测的异常。
    5.2)运行时异常，其中Exception有一个特殊的子类RuntimeException，以及RuntimeException的子类是运行异常，也就是说这个异常时编译时不被检查的异常。
    6)记住finally很有用，主要用户关闭资源。无论是否发生异常，资源都必须进行关闭。System.exit(0);//退出jvm，只有这种情况finally不执行。
    7)常见异常：
    7.1)脚标越界异常(IndexOutOfBoundsException)
    7.2)空指针异常(NullPointerException)
    7.3)类型转换异常(ClassCastException)
    7.4)没有这个元素异常(NullPointerException)
    7.5)不支持操作异常
    11.java常见的软件包
    1)java.langlanguage java的核心包，jdk1.2之后，该包中的类被自动导入。
    2)java.awt定义的都是用于java图形界面开发的对象
    3)java.swing提供所有的windows桌面应用程序包括控件
    4)java.net用于java网络编程方面的对象都在该包中
    5)java.ioinput,output用于操作设备上数据的对象都在该包中。比如：读取硬盘数据，往硬盘写入数据
    6)java.utiljava的工具包，时间对象，集合框架
    13.多线程
    1)进程：正在进行中的程序，其实进程就是一个应用程序运行时的内存分配空间。
    2)线程：其实就是进程中一个程序执行控制单元，一条执行路径。进程负责的是应用程序的空间标示，线程负责的是应用程序的执行顺序。一个进程至少有一个线程在运行，当一个进程中出现多个线程，就称这个应用程序是多线程应用程序。每个线程在栈区中都有自己的执行空间，自己的方法区，自己的变量。JVM在启动的时候，首先有一个主线程，负责程序的执行，调用的是main函数。主线程执行的代码都在main方法中。当产生垃圾时，收集垃圾的工作，是不需要主线程来完成。因为这样会出现主线程中的代码执行停止，效率较低，所以由单独一个线程来负责垃圾回收。
    3)随机性原理：因为cpu的快速切换造成，哪个线程获取到了cpu的执行权，哪个线程就执行。
    3.1)返回当前线程的名称:Thread.currentThread().getName()。
    3.2)线程的名称是由：Thread-编号定义的，编号从0开始。
    3.3)线程要运行的代码都统一存放在了run方法中。线程要运行必须要通过类中指定的方法开启，start方法。
    3.4)创建线程的第一种方式：继承Thread，由子类复写run方法
    3.5)步骤：
    3.5.1)定义类继承Thread方法。
    3.5.2)目的是复写run方法，将要让线程运行的代码都存储到run方法中。
    3.5.3)通过创建Thread类的子类对象，创建线程对象。
    3.5.4)调用线程的start方法，开启线程，并执行run方法。
    3.6)创建线程的第二种方式：实现一个接口Runnable
    3.6.1)步骤：
    3.6.2)定义类实现Runnable接口
    3.6.3)覆盖接口中的run方法(用于封装线程要运行的代码)
    3.6.4)通过Thread类创建线程对象
    3.6.5)将实现了Runnable接口的子类对象作为实际参数传递给Thread类中的构造函数
    3.6.6)调用Thread对象的start方法。开启线程，并运行Runnable接口子类的run方法。
    3.6.7)举例:Ticket t = new Ticket();Thread t1 = new Thread(t);t1.start();
    3.6.8)为什么要有Runnable接口的出现？通过继承Thread类的方式，可以完成多线程的建立。但是这种方式有一个局限性，如果一个类已经有了自己的父类，就不可以继承Thread类，因为java单继承的局限性。
    3.7)线程的状态：
    3.7.1)被创建：start()；
    3.7.2)运行：具备执行资格，同时具备执行权；
    3.7.3)冻结：sleep(time),wait() - notifry()唤醒；线程释放了执行权，同时释放执行资格；
    3.7.4)临时阻塞状态：线程具备cpu的执行资格，没有cpu的执行权。
    3.7.5)消亡：stop()

      约瑟夫问题

      昨天做笔试题的时候遇到这个问题，觉得有很意思，在这里记录一下。
      约瑟夫问题：

      • 所有人围成一圈

      • 顺时针报数，每次报到q的人将被杀掉

      • 被杀掉的人将从房间内被移走

      • 然后从被杀掉的下一个人重新报数，继续报q，再清楚，直到剩余一人


        • 输入：一共有n个人，每次报到k的时候，那个人被杀掉
        初始情况：0,1,2…n-2,n-1(共n个人)
        假设第一个人已经自杀，剩下的n-1个人组成了一个新的约瑟夫环(以编号为k%n)的人开始：k k+1 k+2 … n-2 n-1 0 1 2 … k-3 k-2 对剩余的人编号做一下转换：


        x –> x’ (这里的x表示的是最后留下来那个人x’就是我们要求的结果)
        k –> 0
        k+1 –> 1
        …
        k-2 –> n-2
        k-1 –> n-1


                变换之后就成了n-1个人报数的子问题，假如我们知道这个子问题的解(也就是x的坐标),那么我们就可以知道原问题的解。可以知道转换公式为:

        x(n) = (x(n-1) + k)%n

                根据上面的公式，我们可以从只有一个人开始坐标转换，最后到第n轮该人的坐标就是我们要的结果。
