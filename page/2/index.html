图论-最短路

之前就想对常见的图论的算法进行一个总结，借着这个机会开一个坑。

常见的最短路算法

floyd 弗洛伊德算法

思想：想知道a到b点的最短路，只能加入中间节点来缩短路径。那么我们就要遍历所有可能的中间点，才能得到a到b的最短路径。
核心代码只有5行

for(int mid = 0;mid < N;mid++){
    for(int i=0;i < N;i++){
        for(int j=0;j < N;j++){
        if(map[i][j] > map[i][k] + map[k][j])    map[i][j]=map[i][k]+map[k][j];
        }
    }
}

这种算法可以找多源多汇最短路。
时间复杂度 O(N^3)
空间复杂度 O(N^2)

dijkstra 狄杰斯特拉 － 邻接矩阵

思想：这个算法只能计算单源最短路，而且不能计算负权值，这个算法是贪心思想。
e[N][N]邻接矩阵 dis[N]初始点到个点的距离 book[N]标记该点最短路是否找到 假设每个量都被正确初始化了。

for(int times=1;times < N;times++){
    int min = Integer.MAX_VALUE;
    int index = -1;
    //寻找本次最短路，标记为最终最短路
    for(int i=0;i < N;i++){
        if(!book[i] &&dis[i] < min){
            min = dis[i];
            index = i;
        }
    }
    if(index == -1)    break;//提前退出
    book[index] = true;
    for(int j=0;j < N;j++){
        if(e[i][j] == Integer.MAX_VALUE){
            if(dis[j] > dis[index] + e[index][j])    dis[j] = dis[index] + e[index][j];
        }
    }
}

dijkstra 狄杰斯特拉 － 邻接表 ＋ 优先队列

例题：leetcode－744
输入：
times[i] = (u, v, w) u源点v汇点w权重
N 网络节点总数
K 起始点
时间复杂度 O(N^2)
空间复杂度 O(N logN)

public int networkDelayTime(int[][] times,int N,int K){
    //开空间,用邻接表的方式存储图
    Map<Integer,List<int[]>> graph = new HashMap<>();
    //优先队列 <dist , end node>
    PriorityQueue<int[]> heap = new PriorityQueue<int[]>((info1,info2)->info1[0] - info2[0]);
    //距离矩阵 <end node , dist>
    Map<Integer,Integer> dist = new HashMap<>();
    //存入数据
    for(int[] edge: times){
        if(!graph.containsKey(edge[0])){
            graph.put(edge[0],new ArrayList<int[]>());
        }
        graph.get(edge[0]).add(new int[]{edge[1],edge[2]});
    }
    //存入初始点，从K点出发，
    heap.offer(new int[]{0,K});
    while(!heap.isEmpty()){
        int[] info = heap.poll();
        int d = info[0], node = info[1];
        //如果包含这个点continue | 先到先的,因为优先队列能够保证最后的结果是
        if(dist.containsKey(node))    continue;
        //如果出现新的点，放入最后的距离队列
        dist.put(node,d);
        //如果图中包含该点到其他点的边
        if(graph.containsKey(node)){
        //对于出发点的所有边
            for(int[] edge : graph.get(node)){
                int nei = edge[0], d2 = edge[1];
                //如果dist中不包含这个尾部节点
                if(!dist.containsKey(nei))
                    heap.offer(new int[]{d+d2,nei});
            }
        }
    }
    if(dist.size() != N)    return -1;
    int ans = 0;
    for(int cand : dist.values())
    ans = Math.max(ans,cand);
    return ans;
}

Hadoop Streaming

Hadoop streaming是Hadoop的一个工具，它帮助用户创建和运行一类特殊的map/reduce作业，这些特殊的map/reduce作业是由一些可执行文件或者脚本文件充当mapper或者reducer。例如：

Streaming工作原理

在上面的例子里，mapper和reducer都是可执行文件，它们从标准输入读入数据(一行一行读)，并把计算结果发给标准输出。Streaming工具会创建一个Map/Reducer作业，并把它发送给合适的集群，同时监视这个作业的整个执行过程。
如果一个可执行文件被用于mapper，则在mapper初始化时，每个mapper任务会把这个可执行文件作为一个单独的进程启动。mapper任务运行时，它把输入切分成行并把每一行提供给可执行文件进程的标准输入。同时，mapper收集可执行文件标准输出的内容，并把收到的第一行内容转化成key/value对，作为mapper的输出。默认情况下，一行中第一个tab之前的部分作为key，之后的（不包括tab）作为value。如果没有tab，整行作为key值，value值为null。不过，这可以定制，在下文中将会讨论如何自定义key和value的切分方式。
如果一个可执行文件被用于reducer,每个reducer任务会把这个可执行文件作为一个单独的进程启动。Reducer任务运行时，它把输入切分成行并把每一行提供给可执行文件进程的标准输入。同时，reducer收集可执行文件进程标准输出的内容，并把每一行内容转化为key/value对，作为reducer的输出。默认情况下，一行中第一个tab之前的部分作为key，之后的（不包括tab）作为value。在下文中将会讨论如何自定义key和value的切分方式。
以上是Map/Reducer框架和Streaming mapper/reducer之间的基本通信协议。
用户也可以java类作为mapper或者reducer。
用户可以设定stream.non.zero.exit.is.failure true或者false来表明streaming task的返回值非零时是Failure 还是 Success。默认情况，streaming task的返回值非零时是Failure.
